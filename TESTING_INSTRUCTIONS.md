# ИНСТРУКЦИИ ПО ТЕСТИРОВАНИЮ ПРИЛОЖЕНИЯ bary3

## Запуск приложения на эмуляторе

Приложение уже запущено на эмуляторе `emulator-5554`. Если нужно перезапустить:

```bash
flutter run -d emulator-5554
```

## План тестирования

Подробный план тестирования находится в файле `TESTING_REPORT.md`. Ниже краткая инструкция по основным проверкам.

### 1. БАЗОВАЯ ПРОВЕРКА ЗАПУСКА
- ✅ Приложение запускается без ошибок
- ✅ Отображается главный экран (Баланс)
- ✅ Видны все 5 вкладок внизу

### 2. НАВИГАЦИЯ
Проверьте переключение между вкладками:
- Баланс → Копилки → Календарь → Уроки → Настройки
- Переключение должно быть плавным
- Состояние экранов должно сохраняться

### 3. ЭКРАН БАЛАНС
**Основные проверки:**
1. Отображение текущего баланса
2. Кнопка "Добавить" → добавление дохода
3. Кнопка "Потратить" → добавление расхода
4. Кнопка "Запланировать" → создание события
5. Фильтры: День/Неделя/Месяц/Всё
6. Переключатель "Прогноз баланса"
7. Кнопка "Инструменты"

**Тестовый сценарий:**
- Добавьте доход 100 EUR
- Добавьте расход 30 EUR
- Проверьте, что баланс обновился (70 EUR)
- Проверьте транзакции в списке

### 4. ЭКРАН КОПИЛКИ
**Основные проверки:**
1. Создание новой копилки
2. Просмотр деталей копилки
3. Пополнение копилки
4. Снятие из копилки

**Тестовый сценарий:**
- Создайте копилку "Тест" на 200 EUR
- Пополните на 50 EUR
- Проверьте прогресс (25%)
- Снимите 20 EUR
- Проверьте обновление прогресса

### 5. ЭКРАН КАЛЕНДАРЬ
**Основные проверки:**
1. Отображение календаря
2. Создание запланированного события
3. Просмотр событий дня
4. Выполнение события

**Тестовый сценарий:**
- Создайте событие на завтра (расход 40 EUR)
- Проверьте маркер на календаре
- Выполните событие
- Проверьте создание транзакции

### 6. ЭКРАН УРОКИ
**Основные проверки:**
1. Загрузка уроков
2. Прохождение урока
3. Прохождение квиза
4. Получение XP

**Тестовый сценарий:**
- Откройте первый бесплатный урок
- Пройдите контент
- Ответьте на вопросы квиза
- Проверьте начисление XP

### 7. ЭКРАН НАСТРОЙКИ
**Основные проверки:**
1. Изменение языка (RU/DE/EN)
2. Изменение темы (Blue/Purple/Mint)
3. Изменение валюты (EUR/USD/CHF)
4. Настройки Бари
5. Родительская зона

**Тестовый сценарий:**
- Переключите язык на DE
- Проверьте изменение интерфейса
- Переключите тему на Purple
- Проверьте изменение градиента

### 8. СИСТЕМА BARI
**Основные проверки:**
1. Overlay Бари на экранах
2. Чат с Бари
3. Рекомендации Бари
4. Реакции на действия

**Тестовый сценарий:**
- Добавьте доход → проверьте реакцию Бари
- Откройте чат с Бари
- Задайте вопрос о копилках
- Проверьте ответ

### 9. ИНСТРУМЕНТЫ
**Основные проверки:**
1. Открытие ToolsHubScreen
2. Доступ к калькуляторам
3. Использование калькулятора

**Тестовый сценарий:**
- Откройте Инструменты
- Выберите калькулятор "Бюджет 50/30/20"
- Введите данные
- Проверьте результат

### 10. РОДИТЕЛЬСКАЯ ЗОНА
**Основные проверки:**
1. Вход по PIN
2. Просмотр статистики
3. Одобрение транзакций

## Автоматизированные тесты

Для запуска базовых тестов:

```bash
flutter test
```

Для запуска интеграционных тестов:

```bash
flutter test test/integration_test.dart
```

## Отчёт о тестировании

Заполняйте результаты тестирования в файле `TESTING_REPORT.md`, отмечая выполненные тесты и найденные проблемы.

## Известные ограничения

- Unit-тесты не могут полностью проверить UI-взаимодействия
- Для полноценного тестирования навигации и переходов рекомендуется ручное тестирование или использование `integration_test` пакета
